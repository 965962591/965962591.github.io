hexo博客的一些问题

发表于 2020-11-27 分类于 hexo
本文字数： 1.1k 阅读时长 ≈ 1 分钟
记录一次部署静态博客的遇到的一些问题，去更好的保存我想写的内容。

常见问题

  1. 图片显示
  2. 多标签编写
  3. 博客草稿修改
  4. abbrlink插件
  5. yilia主题头像设置
1.图片显示

1、安装**CodeFalling/hexo-asset-image**插件

npm install hexo-asset-image --save

2、在_config.yml配置文件中修改**post_asset_folder**属性值

post_asset_folder: true

3、运行命令，使用新的配置

#清除就配置文件生成的项目
hexo clean
#用新的配置文件生成项目
hexo g

注意：新配置会自动在source\_posts文件夹下生成文章的同名文件夹，需要将你使用的图片存放在文章同名文件夹中即可

4、文章中使用图片

![](文章同名文件夹/图片名.jgp)
2.多标签编写

在默认生成的头文件中编写

单标签：

title:
date: 
tags: hexo博客
categories: hexo

多标签：

tags: [标签1, 标签2, 标签3]

注意：1、需要使用英文格式的逗号+空格

​ 2、tags:后面要加空格（hexo的配置文件尤其不能忘记）

3.博客草稿修改问题

我们使用hexo n生成新的博客，会使用默认的草稿文件来生成头文件，但是包含的内容比较少，我们可以按需添加

修改

目录：\scaffolds\post.md

title: &#123;&#123; title &#125;&#125;
date: &#123;&#123; date &#125;&#125;
tags:
categories:
4.abbrlink插件

1、安装插件

npm install hexo-abbrlink --save

2、修改_config.yml配置文件

permalink: posts/:abbrlink/
abbrlink:
  alg: crc32
  rep: hex

注意：不要修改为permalink: posts/abbrlink.html/或者

permalink: posts/abbrlink/都会出现问题，前者会出现生成的文章访问不到图片，

后者会出现所有的文章的链接都是一样的，异常情况。

5.yilia主题头像设置

1、主题配置文件目录

目录：\themes\yilia\_config.yml

2、存放头像的目录

目录：\themes\yilia\source\assets

没有assets文件夹的，新建一个，将图片放进去。

3、修改配置文件

#你的头像url
avatar: /assets/1.JPG

注意：1、avatar:后面的空格不要忘记了

​ 2、要使用/assets/图片名.图片格式，/assets前面的/不能忘记加
